Standards

On this page we have outlined some basic standards that all submitted resources should follow. These standards may evolve in the future if problems are found or more efficient routines are established.

Scripts

  • All scripts must contain or come with a readme outlining specifics to operation such as limitations or caveats to watch out for.
  • llSleep should be avoided as it suspends the entire script thread rather than just the script. Use timers instead.
  • Blocks of code should be kept short and should finish executing well before the 30 second event timeout.
  • Loops have the capability of running an event into timeout, consider using different methods of looping to avoid the timeout.
  • States should be avoided and only used if the context does not allow other methods.
  • Listeners should be avoided where possible and if absolutely required should only be created temporarily and removed if no longer needed. See llListenRemove
  • Where possible standard LSL functions should be used along with standard LSL events. If necessary OSSL functions may be used, but a disclaimer and information on the required permissions should be included with the script or the readme.

    • Buildings

  • All submitted buildings and building parts must be fully licensed for the structure, scripts and any texture applied to it. When in doubt, do not supply textures or scripts and provide reference to original designs or parts.
  • Buildings should be a single linkset or come with instructions on how to assemble multiple linksets. Alignment indicators or obvious connections should be described in the readme. Avoid using rezzers.
  • Land Impact information along with count of unique textures should be included in the readme.
  • Included scripts should be provided as copy along with the structure containing them. Information on which Primitives contain scripts should be included in the readme.
    • These standards are subject to change and may be changed at any point.
